Swim Aus Database / SwimCentral

Competition Upload
Results via Swim Central · short-term CSV · long-term national automation
Retention & Pathway Data
Stage progression, drop-off signals & re-engagement tracking
HP Tools — SPARTA II
Biomechanics & technique analysis · gated by subscription tier
Manual Coach Input
Session notes, technique flags & assessments
all source data normalised into platform
Layer 2 Core Platform — Data spine & intelligence
Unified Swimmer Identity
Single persistent profile per athlete across clubs, states & systems
Event-Driven Pipeline
Data events trigger downstream analytics & notifications automatically
Data Normalisation
Reconciles formats across all source systems into one canonical model
Analytics Engine
Benchmarks, improvement signals & national pathway scores
Dropout Risk Model
Detects disengaging athletes early — surfaced to coaches, not federation
profiles, benchmarks & signals delivered to users

API Layer

subscription access, tier gating & export requests

Infrastructure & prototype stack

Security posture, Privacy Act alignment, and the technologies that power the current AusSWIM (SWIMAUS) prototype.

Infrastructure

Cloud Native
Role-Based Access Control Swimmer · parent · coach · club admin · state admin · national admin
Privacy Act Compliance Athlete data handled per Privacy Act obligations · access scoped strictly by role · minor data never exported

Prototype stack (SWIMAUS)

  • React 19 + TypeScript — UI components and app logic
  • Vite 7 — build tooling · Tailwind CSS v4 — styling · React Router DOM v7 — routing
  • Recharts — data visualisation · Lucide React — icons
  • Data layer — local mock modules + React context; no live backend in this prototype
